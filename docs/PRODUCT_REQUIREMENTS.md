# Product Requirements

Короткое описание общей цели и базовых требований продукта для SDD.

## Product Goal

Продукт - персональный финансовый трекер для учета денежного потока пользователя.

Главная цель - дать максимально быстрый и понятный способ добавления транзакций и мгновенного просмотра последних операций без перегруза аналитикой, сложной навигацией и dashboard-heavy UI.

## Core Value

Приложение должно ощущаться как быстрый инструмент ввода, а не как сложная финансовая система.

Основной сценарий:

- выбрать счет
- добавить транзакцию за несколько секунд
- сразу увидеть ее в списке последних операций

## Core Principles

- `input-first UX` - ввод транзакции является главным сценарием продукта
- быстрый доступ к последним транзакциям сразу на главном экране
- аналитика, полная история и расширенные данные вторичны и вынесены на отдельные страницы
- навигация минимальна и не отвлекает от основного сценария
- интерфейс должен быть чистым, быстрым и понятным

## Functional Requirements

### Transactions

- пользователь может создавать транзакции трех типов: `income`, `expense`, `transfer`
- транзакция должна содержать:
  - счет
  - сумму
  - категорию
  - название или комментарий
  - дату и время
- дата и время опциональны; по умолчанию подставляются текущие дата и время
- валюта не выбирается в каждой транзакции отдельно; она задается в настройках пользователя
- для `transfer` должен поддерживаться перевод между счетами

### Categories

- в системе должен быть стартовый набор категорий
- пользователь может создавать свои категории
- пользовательская категория должна содержать название и иконку

### Main Screen

- на главной странице должны быть доступны:
  - быстрый выбор счета
  - быстрый ввод транзакции
  - список последних транзакций
- последние транзакции должны отображаться сразу после блока ввода

### Navigation

- на старте навигация должна быть минимальной
- основной способ навигации - `Cmd+K` и быстрые действия
- полная история, аналитика, счета, категории и другие расширенные разделы должны быть вынесены на отдельные страницы

# Locales and Currencies
 - поддержка нескольких языков и валют должна быть предусмотрена в настройках
 - язык и валюта должны применяться глобально, а не на уровне отдельных транзакций
 - дефолтные категории должны быть локализованы в зависимости от выбранного языка
 - при смене языка интерфейс должен обновляться без перезагрузки страницы
 - при смене валюты все суммы должны отображаться в новой валюте с учетом текущего курса (если реализовано)
 - языки Английский и Русский должны быть доступны на старте, с возможностью добавления других в будущем
 - валюта по умолчанию - USD, с поддержкой основных валют (EUR, RUB) на старте

## UX Requirements

- добавление транзакции должно занимать минимум действий и времени
- основной экран не должен использовать модальные сценарии для базового ввода транзакции
- ввод транзакции должен быть главным визуальным акцентом страницы
- список последних операций должен быть простым и читаемым
- навигация должна быть вторичной по отношению к сценарию ввода

## UI Direction

- desktop web app
- минималистичный современный SaaS-style интерфейс
- визуально ближе к Linear / Notion, чем к классическому банковскому dashboard
- без sidebar как основного элемента навигации
- без графиков и аналитики на главном экране
- `Cmd+K` palette выступает как power-user navigation layer, а не как основной контент

## Out of Scope For Main Screen

- глубокая аналитика
- полная история с расширенными фильтрами
- сложный dashboard
- перегруженная навигация
