# Task 001 - Core Foundation And Domain

## Status

done

## Product Requirements Covered

- базовая архитектура приложения для персонального финансового трекера
- доменные сущности: транзакции, счета, категории, настройки
- поддержка типов транзакций `income`, `expense`, `transfer`
- глобальные настройки языка и валюты

- [x] Добавлены отдельные stores для счетов, транзакций, категорий и настроек.
- [x] Состояние stores сохраняется в `localStorage` через `@vueuse/core`.
- [x] Добавлены базовые типы `Account`, `Transaction`, `Category`, `Settings`.
- [x] Добавлены стартовые категории и базовая конфигурация приложения.

## Domain Decisions

- [x] `Transaction` реализован как discriminated union для `income`, `expense`, `transfer`.
- [x] Для `income` и `expense` обязательны `accountId` и `categoryId`.
- [x] Для `transfer` обязательны `fromAccountId` и `toAccountId`.
- [x] `transfer` не использует категорию, так как это перемещение средств между счетами, а не доход или расход.
- [x] Для сортировки и отображения операций используется `occurredAt` как время фактической операции.
- [x] `createdAt` остается техническим полем момента создания записи.
- [x] Текущий баланс счета вычисляется из стартового баланса, ручной корректировки и влияния транзакций.

## Acceptance Criteria

- [x] Модель данных полностью покрывает сценарии `income`, `expense`, `transfer`.
- [x] Для `income` и `expense` транзакция содержит счет, сумму, категорию, комментарий и время операции.
- [x] Для `transfer` транзакция содержит счет-источник, счет-получатель, сумму, комментарий и время операции.
- [x] Настройки языка и валюты остаются глобальными сущностями, а не свойствами отдельной транзакции.
- [x] Store отфильтровывает невалидные записи из `localStorage` и работает только с корректными сущностями.
- [x] Транзакции не могут ссылаться на несуществующие счета и категории.
- [x] Новая доменная модель без дополнительных доработок поддерживает UI-реализацию dashboard и transaction flow.
