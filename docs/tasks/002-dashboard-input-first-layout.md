# Task 002 - Dashboard Input First Layout

## Status

partially done

## Product Requirements Covered

- `input-first UX`
- быстрый выбор счета на главном экране
- быстрый ввод транзакции на главном экране
- список последних транзакций сразу под блоком ввода
- минималистичный desktop web интерфейс без dashboard-heavy подхода

## Implementation Steps

- [ ] Перестроить `DashboardPage.vue` вокруг основного пользовательского сценария, а не вокруг заглушки с заголовком.
- [ ] Добавить компактный блок быстрого выбора счета в верхней части главного экрана.
- [ ] Разместить форму ввода транзакции как главный визуальный акцент страницы.
- [ ] Разместить блок последних операций сразу после формы ввода без дополнительных переходов.
- [ ] Убрать с главной страницы элементы, которые не относятся к быстрому вводу и просмотру последних операций.
- [ ] Проверить, что layout работает на desktop и не разваливается на узких экранах.
- [ ] Привести визуальный стиль ближе к minimal SaaS UI, а не к банковскому dashboard.

## Logical Chunks

- [ ] Собрать структуру главного экрана.
- [ ] Реализовать секцию выбора счета.
- [ ] Подключить секцию быстрого ввода.
- [ ] Подключить секцию последних операций.
- [ ] Провести визуальную шлифовку и адаптацию.

## Acceptance Criteria

- [ ] Главная страница сразу ведет пользователя к вводу транзакции.
- [ ] Выбор счета, ввод транзакции и последние операции доступны на одном экране.
- [ ] На главной странице нет графиков, сложной аналитики и перегруженной навигации.
- [ ] Базовый сценарий не требует открытия модальных окон.
