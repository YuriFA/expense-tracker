# Stage 5. Dashboard

## Goal

Собрать главный экран, который быстро показывает финансовую сводку по данным пользователя и помогает перейти к ключевым действиям.

## Context

Из `PROJECT_STRUCTURE.md` dashboard должен собираться как набор feature-компонентов в `components/dashboard/`, а сложные агрегации лучше держать в `utils/` или `composables/`, а не внутри шаблонов.

Из `STORE_SCHEMA.md` основным источником данных будет `transactionsStore`, а информация о категориях должна подтягиваться через `categoriesStore`.

## What To Build

- Страницу dashboard с финансовой сводкой.
- Набор карточек и виджетов для ключевых показателей.
- Быстрые действия для создания транзакций.
- Надежные вычисления текущего баланса и месячных агрегатов.

## Detailed Tasks

1. Создать `DashboardPage.vue`, которая собирает экран из dashboard-компонентов.
2. Реализовать `BalanceCard` для отображения общего баланса.
3. Реализовать `IncomeExpenseCard` или два отдельных виджета для:
   - доходов за текущий месяц
   - расходов за текущий месяц
4. Реализовать `RecentTransactions` с несколькими последними операциями.
5. Реализовать `CategorySpendingCard` для топа категорий по расходам.
6. Определить, какие вычисления стоит вынести в `useMonthlyStats` или в чистые `utils`:
   - фильтрация по текущему месяцу
   - суммирование доходов и расходов
   - выбор последних транзакций
   - агрегация расходов по категориям
7. Подтянуть названия и метаданные категорий через `categoriesStore`.
8. Добавить быстрые действия для создания дохода и расхода.
9. Определить UX быстрых действий:
   - переход на страницу транзакций
   - открытие формы сразу в нужном типе
10. Предусмотреть состояние экрана при малом количестве данных или полном отсутствии транзакций.
11. Убедиться, что dashboard автоматически обновляется после CRUD-операций над транзакциями.

## Dependencies

- Требует готового `transactionsStore`, `categoriesStore` и базового CRUD транзакций.
- Может использовать composables из списка проекта, например `useMonthlyStats`.
- Будет переиспользовать часть логики аналитики, поэтому агрегации лучше делать централизованно.

## Acceptance Criteria

- На `/` отображается dashboard со сводкой.
- Баланс считается корректно как `income - expense`.
- Доходы и расходы за текущий месяц отображаются корректно.
- Видны последние транзакции.
- Видны топ-категории по расходам.
- Быстрые действия реально ускоряют создание новой транзакции.
- При пустых данных экран остается понятным и полезным.

## Notes

- Не стоит дублировать одну и ту же бизнес-логику одновременно в dashboard и analytics.
- Если часть карточек требует одинаковых фильтров по текущему месяцу, лучше вынести это в общую вычислительную функцию.
- Карточки должны быть простыми отображающими компонентами без прямой записи в `localStorage` и без сложной доменной логики внутри.
