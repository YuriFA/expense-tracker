# Stage 6. Analytics

## Goal

Добавить экран аналитики, который строит понятные графики и summary-метрики по реальным данным транзакций, а не по статическим заглушкам.

## Context

Из `PROJECT_STRUCTURE.md` аналитика должна быть собрана из компонентов в `components/analytics/`, а вычисления лучше держать вне визуальных компонентов.

Из `STORE_SCHEMA.md` `transactionsStore` и `categoriesStore` дают базу для построения агрегатов по месяцам и категориям.

## What To Build

- Отдельную страницу аналитики.
- Интеграцию с одной библиотекой графиков.
- Графики и summary-метрики по транзакциям.
- UI фильтра периода.
- Fallback-состояния для пустых или слишком маленьких выборок.

## Detailed Tasks

1. Выбрать библиотеку графиков из рекомендуемых в проекте:
   - `Chart.js`
   - `ECharts`
2. Зафиксировать критерии выбора:
   - простота интеграции с Vue 3
   - достаточность для pie и line/bar сценариев
   - умеренная сложность для pet-проекта
3. Создать `AnalyticsPage.vue`.
4. Реализовать `AnalyticsFilters` для выбора периода.
5. Реализовать `SpendingByCategoryChart` для круговой диаграммы расходов по категориям.
6. Реализовать `MonthlyTrendChart` для доходов и расходов по месяцам.
7. Реализовать `SummaryStats` с дополнительными показателями:
   - средние траты за период
   - самая затратная категория
   - дополнительные агрегаты по необходимости
8. Вынести расчеты в `utils/` или composables:
   - группировка транзакций по месяцам
   - суммирование расходов по категориям
   - расчет среднего значения
   - поиск максимальной категории расходов
9. Убедиться, что в аналитику попадают только релевантные транзакции выбранного периода.
10. Сделать fallback UI для случаев:
   - нет транзакций
   - слишком мало данных для содержательного графика
   - есть только доходы или только расходы
11. Проверить читаемость графиков в сочетании с цветовыми метками категорий.

## Dependencies

- Требует завершенных stages 2, 3 и желательно 5.
- Использует `transactionsStore`, `categoriesStore`, а позже может пересекаться с `settingsStore` для локали и валюты.
- После выбора chart library нужно учитывать ее в дальнейшей поддержке и оформлении темы.

## Acceptance Criteria

- На странице аналитики есть рабочие графики по реальным данным.
- Круговая диаграмма расходов по категориям строится корректно.
- График доходов и расходов по месяцам строится корректно.
- Фильтр периода влияет на все метрики и графики экрана.
- Показываются средние траты и самая затратная категория.
- При недостатке данных экран показывает понятный fallback, а не сломанный график.

## Notes

- Для pet-проекта лучше выбрать одну библиотеку и не добавлять абстракцию под несколько решений без необходимости.
- Если логика агрегации будет использоваться и в dashboard, ее нужно держать в общем месте.
- Важно не смешивать форматирование данных для UI и сами расчеты: сначала считаем, потом форматируем.
