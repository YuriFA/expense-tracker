# Stage 7. i18n

## Goal

Сделать приложение двуязычным (`ru/en`) так, чтобы переключение языка влияло на интерфейс, даты и формат отображения чисел без точечных хаков в отдельных компонентах.

## Context

Из `PROJECT_STRUCTURE.md` конфигурация `i18n` должна лежать в `src/app/i18n/`, а тексты не должны оставаться захардкоженными в route pages и feature-компонентах.

Из `STORE_SCHEMA.md` `settingsStore` должен синхронизироваться с `i18n`, потому что хранит `locale` пользователя.

## What To Build

- Конфигурацию `vue-i18n` с двумя локалями.
- Файлы переводов для `ru` и `en`.
- Переключатель языка в настройках или layout.
- Локализованное форматирование дат и валют.
- Проверку pluralization и согласованности текстов.

## Detailed Tasks

1. Подключить и настроить `vue-i18n` на уровне `src/app/i18n`.
2. Создать локали:
   - `src/app/i18n/locales/ru.ts`
   - `src/app/i18n/locales/en.ts`
3. Организовать ключи переводов так, чтобы они масштабировались по фичам:
   - navigation
   - dashboard
   - transactions
   - analytics
   - settings
   - common actions
4. Найти и вынести все пользовательские строки из компонентов и страниц в переводы.
5. Реализовать `LocaleSwitcher`.
6. Связать переключение языка с `settingsStore.setLocale(locale)`.
7. Обеспечить обновление `vue-i18n` при изменении значения в store.
8. Локализовать даты через общий composable или helper, а не точечно в каждом компоненте.
9. Локализовать формат валюты с учетом `locale` и выбранной `currency`.
10. Проверить pluralization там, где есть счетчики, например количество транзакций, категорий или записей.
11. Проверить интерфейс целиком в двух языках и убедиться, что тексты не выпадают из layout.

## Dependencies

- Требует базового `i18n` подключения из Stage 1.
- Зависит от `settingsStore` из Stage 2.
- Пересекается с этапом настроек, где пользователь будет менять язык и валюту.

## Acceptance Criteria

- Приложение поддерживает `ru` и `en`.
- Все пользовательские строки вынесены в словари переводов.
- Язык можно переключить через UI.
- Переключение языка действительно обновляет интерфейс без перезагрузки.
- Даты и денежные значения форматируются с учетом выбранной локали.
- Pluralization работает корректно в нужных местах.

## Notes

- Не стоит хранить уже переведенные строки в store или в данных доменной модели.
- Лучше заранее согласовать структуру translation keys, чтобы потом не пришлось массово переименовывать их по мере роста проекта.
- Если в графиках и summary используются подписи, их тоже нужно вести через i18n.
